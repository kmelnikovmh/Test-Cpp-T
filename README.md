# Alice Sandbox Calc
Консольное C++ приложение — симулятор бота для игры-песочницы.
Программа принимает описание подземелья из файла, запускает двухфазный алгоритм бота
(исследование + возвращение) и записывает последовательность действий в `result.txt`.
При некорректных входных данных в `result.txt` записывается строка с ошибкой.

Проект выполнен в учебных целях.


## Основные возможности
- строгая валидация входных данных с указанием ошибочной строки
- симуляция действий бота с подсчётом собранных ресурсов и общей ценности
- расширяемость: интерфейс `IBot` позволяет подключить любой другой алгоритм
- реализованный двухфазный алгоритм бота с BFS навигацией - исследование и возвращение


## Архитектура проекта
Проект логически разделён на пять уровней:
- `Parser` — проверяет и парсит входные данные (формат, диапазоны, симметричность рёбер). Не содержит бизнес-логики. Возвращает `variant<ParsedInput, string>` — либо корректный ввод, либо строку с ошибкой
- `Resource` — перечисление ресурсов (iron, gold, gems, exp), базовые ценности и функция удвоения целевого ресурса
- `Dungeon` — хранит граф комнат (оригинал + мутабельная копия), предоставляет BFS-методы для поиска кратчайшего пути и ближайшей непосещённой комнаты. `reset()` восстанавливает состояние из оригинала
- `IBot / AliceBot` — интерфейс бота и его реализация. `AliceBot::run()` принимает `Dungeon&`, запас еды и целевой ресурс, возвращает `vector<Action>`. Действия представлены через `variant<GoAction, CollectAction>`
- `Simulator` — воспроизводит действия бота на чистом подземелье, формирует текстовый вывод с состояниями комнат и итоговой строкой `result`

Краткий поток исполнения:
`main` → `parse_input` → `AliceBot::run` → `Dungeon::reset` → `simulate` → `result.txt`

## Сборка

### Зависимости и CI
- C++20
- CMake ≥ 3.19

Для обычной сборки дополнительные тестовые зависимости не нужны.
При включённом `BUILD_TESTING=ON` проект использует GoogleTest v1.14.0
(скачивается автоматически через `FetchContent`, если не найден в системе).

CI сборка:
- Linux: GCC, Debug + ASan/UBSan
- macOS: Clang, Debug (без санитайзеров)
- Windows: MSVC 17 2022

### Запуск
Сборка из корня репозитория:

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
```

```bash
cmake --build build
```

Запуск с входным файлом `in.txt`:

```bash
./build/alice-sandbox-calc in.txt
```

Результат записывается в `result.txt` в рабочей директории

### Тестирование
Тесты собираются как отдельные исполняемые файлы, а также:
- Компиляторные флаги: GCC/Clang `-Wall -Wextra -Wpedantic -pedantic-errors -Werror`, MSVC `/W4 /WX /permissive-`
- В Debug собираются с `-fsanitize=address,undefined` (Linux)
- В Debug проверяются статическим анализатором clang-tidy

Для каждого логического модуля определён отдельный тестовый таргет:
- `resource-test`
- `dungeon-test`
- `parser-test`
- `simulator-test`
- `alice-bot-test`
- `main-test` (интеграционный, запускает бинарник)

Тесты запускаются через CTest с цветным выводом (GTEST_COLOR=1):

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Debug -DBUILD_TESTING=ON
```

```bash
cmake --build build --target execute-tests
```

В CI `BUILD_TESTING=ON` включается явно на этапе `cmake -S . -B build ...`.
